# MsgGo 隐私政策

**最后更新日期：2026年01月02日**

MsgGo（以下简称“本应用”）深知隐私对您的重要性，并致力于保护您的个人数据。本隐私政策旨在向您说明本应用如何处理数据、我们请求权限的原因，以及我们为保护您的隐私所做的努力。

### 1. 核心原则：零网络访问，本地处理
MsgGo 的设计初衷是提供一个完全私密、安全的工具。本应用**未请求外部网络访问权限**。这意味着：
- 您的所有数据（包括 Excel 文件内容、手机号码、短信模板等）**绝不会**被上传到任何服务器或第三方云端。
- 所有的逻辑处理（变量替换、短信生成、发送控制）均完全在您的设备本地完成。

### 2. 我们通过哪些权限使用您的数据
为了实现批量发送短信的核心功能，我们需要请求以下权限：

- **发送短信 (SEND_SMS)**： 核心功能。用于执行您发起的批量短信发送任务。
- **读取电话状态 (READ_PHONE_STATE)**：用于识别多卡设备中的 SIM 卡信息（如卡槽索引、运营商名称），以便您选择特定的 SIM 卡发送短信。
- **读取/写入外部存储 (READ_EXTERNAL_STORAGE)**：用于读取您选择并授权的 Excel 数据文件，以及在您主动导出调试日志时保存文件。
- **前台服务 (FOREGROUND_SERVICE)** 与 **通知权限 (POST_NOTIFICATIONS)**：确保应用在发送大量短信过程中（即使切到后台）不被系统中断，并实时向您反馈发送进度。

### 3. 数据存储与生命周期
- **Excel 数据**：我们会拷贝您的输入数据到缓存目录中用于访问。在清除缓存后，数据即刻失效。
- **本地历史记录**：为了提升您的使用体验，本应用会在本地保存您最近打开的文件路径、短信模板和号码列选择。这些数据存储在应用的私有目录中，您可以随时在“设置”中点击“清除缓存”来彻底移除这些记录。
- **调试日志**：调试日志仅记录应用运行逻辑。只有在您**手动点击**“导出调试日志”并主动分享给开发者时，日志才会被外部访问。

### 4. 第三方服务
MsgGo **不包含**任何第三方分析工具、追踪器、广告 SDK 或 SDK 监控插件。由于没有网络权限，任何形式的第三方数据采集在本应用内均物理失效。

### 5. 您的权利
- **撤回权限**：您可以随时通过系统设置撤回已授予的权限，但这可能会导致相关功能无法使用。
- **删除数据**：您可以通过“设置 - 清除缓存”删除所有本地保存的任务配置缓存。

### 6. 联系我们
由于本应用为开源工具且不具备网络通信能力，如您对隐私政策有任何疑问，请通过设置界面的源代码链接与我们联系。
